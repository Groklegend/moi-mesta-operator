# Мои места — Сайт-помощник оператору (Версия 1)

Веб-приложение для операторов холодного обзвона: быстрый поиск ответов на возражения клиентов. Админка для CEO/ком.директора — управление контентом и статистика.

## Стек
- **Frontend**: чистый HTML / CSS / JS (без сборки), Chart.js для графиков
- **Backend**: Supabase (PostgreSQL + Auth + авто-API)
- **Хостинг**: GitHub Pages (бесплатно)

## Структура проекта

```
sait-operatora/
├── index.html              ← страница оператора
├── styles.css
├── login.html              ← вход админа
├── admin.html              ← админ-панель (рубрики, возражения, шпаргалка, документы)
├── stats.html              ← статистика
├── js/
│   ├── config.js           ← URL и ключ Supabase (заполнить!)
│   ├── supabase-client.js
│   ├── operator.js
│   ├── admin.js
│   └── stats.js
└── sql/
    └── schema.sql          ← выполнить в Supabase SQL Editor
```

## Развёртывание (30 минут)

### 1. Supabase
1. Зарегистрироваться на [supabase.com](https://supabase.com), создать проект (бесплатный tier).
2. В **SQL Editor** выполнить содержимое `sql/schema.sql` — создаст таблицы, RLS-политики и начальные данные.
3. В **Authentication → Users** нажать **Add user** (Invite) и создать одного пользователя с email + паролем — это и будет общий логин админа.
4. В **Authentication → Providers → Email**: отключить «Confirm email» (или подтвердить руками из письма). Для простоты — отключить.
5. В **Project Settings → API** скопировать:
   - `Project URL`
   - `anon public` ключ (не `service_role`!)

### 2. Настройка фронта
Открыть `js/config.js` и вставить:
```js
window.SUPABASE_CONFIG = {
  url: 'https://xxxxxxx.supabase.co',
  anonKey: 'eyJhb...'
};
```

### 3. GitHub Pages
1. Создать публичный GitHub-репозиторий.
2. Закоммитить содержимое папки `sait-operatora/` в корень репо.
3. **Settings → Pages → Source: `Deploy from branch`**, ветка `main`, папка `/` → Save.
4. Через ~30 секунд сайт доступен на `https://<user>.github.io/<repo>/`.

Админка: `/login.html`, статистика: `/stats.html`.

### 4. Наполнение контентом
Зайти в админку → последовательно заполнить:
1. **Рубрики** — Рестораны, Стоматологии, Салоны красоты и т.д.
2. **Возражения** — для каждого отметить «Общее» или выбрать рубрику, указать ключевые слова для поиска.
3. **Шпаргалка** — О продукте, Тарифы, Ключевые отличия, Скрипт приветствия, FAQ.
4. **Документы** — ссылки на КП, презентацию, договор.

## Локальный запуск

Просто откройте `index.html` в браузере. Supabase-клиент работает по HTTPS напрямую, сервер не нужен. Для `http://localhost` — запустите любой статический сервер:
```bash
python3 -m http.server 8000
# → http://localhost:8000
```

## Форматирование в текстах ответов и шпаргалки

- `**жирный**` → **жирный**
- Строка, начинающаяся с `-`, `*` или `•` → пункт маркированного списка
- Пустая строка → перенос абзаца
- Копирование ответа в буфер копирует исходный текст (без HTML) — его можно вставлять в любой мессенджер.
